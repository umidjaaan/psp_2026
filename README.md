## 3. Задание

### 3.1. Основное задание

Создать статический интерфейс веб-приложения, связанного с тематикой **заказа и комплектации наноспутников (CubeSat)**.
Интерфейс должен позволять представить процесс выбора компонентов (системы ориентации, датчики и т.д.), работу с калькулятором и взаимодействие пользователя с системой.

Необходимо:
- реализовать структуру страницы на HTML;
- оформить интерфейс через CSS;
- избегать устаревших подходов (inline-стили);
- обеспечить читаемость и логичность интерфейса.

---

### 3.2. Дополнительное задание

Разработать многостраничный сайт, где каждая страница отвечает за отдельный логический блок:

- `HomePage.html` — вводная информация о проекте CubeSat Enterprise
- `WorkGoalPage.html` — цель работы
- `CalculatorPage.html` — интерфейс калькулятора/расчета компонентов
- `AboutPage.html` — сведения об авторе

Все страницы объединены общей навигационной панелью.

---

## 4. Как реализовано (теория и фрагменты кода)

### 4.1. Семантика и навигация

Для построения структуры используются семантические элементы HTML, что делает код более понятным и логически организованным.

Навигация реализована через верхнюю панель ссылок. Активный раздел (например, "Главная") выделяется стилистически, что помогает пользователю ориентироваться в разделах каталога.

```html
<nav class="navbar">
    <div class="container">
        <a class="navbar-brand text-info" href="./HomePage.html">CubeSat Enterprise</a>
        <ul class="navbar-nav">
            <li class="nav-item">
                <a class="nav-link active" href="./HomePage.html">Главная</a>
            </li>
            <li class="nav-item">
                <a class="nav-link" href="./CalculatorPage.html">Калькулятор</a>
            </li>
            <li class="nav-item">
                <a class="nav-link" href="./WorkGoalPage.html">Алгоритмы (ДЗ 1)</a>
            </li>
        </ul>
    </div>
</nav>
4.2. Организация интерфейса
Контент разбит на отдельные визуальные блоки и карточки компонентов, что делает инженерный интерфейс более строгим и удобным для восприятия.

Каждая страница содержит центральный элемент, в котором отображается основная информация (например, карточка детали).

HTML


<section class="catalog-section">
    <article class="component-card">
        <h2>Система ADCS</h2>
        <span class="badge">Space Ready</span>
        <p>
            Точное наведение спутника на орбите по датчикам звездной ориентации.
        </p>
        <div class="price-tag">2 100 000 ₽</div>
        <button class="btn btn-primary">Подробнее</button>
    </article>
</section>
Основной функциональный блок (интерфейс калькулятора):

HTML


<div class="calculator-container">
    <div id="display" class="calc-display">0</div>
    <div class="calc-buttons">
        <button class="btn-calc action">C</button>
        <button class="btn-calc action">+/-</button>
        <button class="btn-calc action">%</button>
        <button class="btn-calc operator">/</button>
    </div>
</div>
Такая модульная структура позволяет легко масштабировать интерфейс и добавлять новые элементы управления спутником.

4.3. Стилизация и визуальные эффекты
Оформление выполнено в едином "космическом" стиле с использованием темной цветовой схемы (Dark Mode) и неоновых голубых акцентов, характерных для современных технологических панелей.

Применяются эффекты наведения, скругления границ и контрастные переходы.

CSS


body {
  background-color: #12161b;
  color: #e4e6eb;
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
}

.component-card {
  background: #1e252d;
  border: 1px solid #2d3748;
  border-radius: 12px;
  padding: 25px;
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.component-card:hover {
  transform: translateY(-5px);
  box-shadow: 0 8px 24px rgba(0, 168, 255, 0.15);
}

.btn-primary {
  background: #0095ff;
  color: white;
  border: none;
  border-radius: 6px;
  padding: 10px 20px;
  cursor: pointer;
}

.btn-primary:hover {
  background: #0077cc;
}
