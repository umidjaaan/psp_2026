## 3. Задание

### 3.1. Основное задание

Необходимо создать простое веб-приложение с выводом карточек и страницей подробной информации по каждой карточке. Проект должен быть организован в виде одностраничного приложения (SPA), в котором навигация и отображение контента выполняются средствами JavaScript без полной перезагрузки страницы.

Основные требования:
- инициализация проекта через `npm`;
- подключение библиотек через менеджер пакетов;
- разделение кода на логические модули;
- отображение карточек с информацией о космических компонентах;
- открытие подробной страницы по выбранному элементу.

### 3.2. Домашнее задание (Алгоритмика)

В интерфейс приложения необходимо встроить набор алгоритмических задач, адаптированных под предметную область CubeSat Enterprise. В проекте используются задания, связанные с обработкой массивов, объектов и математическими преобразованиями инженерных данных. Каждая задача представлена как отдельный модуль с интеграцией в общий пользовательский интерфейс приложения[cite: 4].

### 3.3. Дополнительное задание

Дополнительно было реализовано оформление интерфейса в корпоративном "космическом" стиле (Dark Mode). Для верстки используется Bootstrap 5 и стандартные утилиты позиционирования, сетки и выравнивания[cite: 4]. Кроме того, в проект интегрирована библиотека Three.js для визуализации интерактивных 3D-моделей на страницах деталей.

## 4. Теория и реализация (Примеры из кода)

### 4.1. Архитектура SPA и компонентный подход

**Теория:**
В одностраничном приложении весь интерфейс отображается внутри одного корневого контейнера[cite: 4]. При переходе между разделами страница не перезагружается, а содержимое корня обновляется программно[cite: 4]. Такой подход упрощает навигацию и позволяет повторно использовать компоненты[cite: 4].

В проекте запуск приложения выполняется через главный модуль, который получает доступ к корневому элементу и инициализирует главную страницу[cite: 4]. Навигация реализована через глобальную функцию `window.navigateTo`.

**Пример из кода:**

```javascript
import MainPage from './pages/main/index.js';
import CalculatorPage from './pages/calculator/index.js';
// ... импорты других страниц

const root = document.getElementById('root');

window.navigateTo = (pageName) => {
    root.innerHTML = ''; // Очистка корневого контейнера

    switch(pageName) {
        case 'main':
            new MainPage(root).render();
            break;
        case 'calculator':
            new CalculatorPage(root).render();
            break;
        // ... роутинг остальных страниц
    }
};

// Инициализация стартовой страницы
window.navigateTo('main');
4.2. Структура страницы и работа с Bootstrap
Теория:

Для интерфейса используется готовая сетка Bootstrap 5, что позволяет быстро организовать адаптивную верстку без написания большого объема собственного CSS[cite: 4]. Вся страница строится вокруг корневого контейнера, а основная область контента занимает свободное пространство между шапкой и подвалом[cite: 4].

В HTML-документе применяется темная тема (bg-dark text-light), настраивается навигационная панель (navbar-dark), а также подключаются внешние скрипты для 3D-графики.

Пример из кода:

HTML


<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>CubeSat Enterprise</title>
    <link href="node_modules/bootstrap/dist/css/bootstrap.min.css" rel="stylesheet">
    <link rel="stylesheet" href="styles.css">
</head>
<body class="bg-dark text-light">

    <nav class="navbar navbar-expand-lg navbar-dark bg-dark border-bottom border-secondary mb-4">
        <div class="container">
            <a class="navbar-brand fw-bold text-info" href="#" onclick="window.navigateTo('main')">CubeSat Enterprise</a>
            <!-- Навигационные ссылки -->
        </div>
    </nav>

    <div id="root"></div> <!-- Контейнер для SPA -->

    <script src="node_modules/bootstrap/dist/js/bootstrap.bundle.min.js"></script>
    <script src="[https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js](https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js)"></script>
    <script type="module" src="main.js"></script>
</body>
</html>
4.3. Хранение данных и рендеринг карточек
Теория:

Данные для карточек вынесены в отдельный модуль[cite: 4]. Это упрощает поддержку проекта, так как контент и логика не смешиваются между собой[cite: 4]. Подобная организация особенно удобна для приложений, где список элементов может изменяться или дополняться[cite: 4].

Компоненты наноспутника (датчики, системы ориентации) хранятся в формате JSON/JS-объектов и динамически рендерятся в интерфейс через методы классов страниц.

Пример из кода:

JavaScript


// Фрагмент рендеринга карточек в MainPage
renderData(data) {
    let html = '<div class="row">';
    data.forEach(item => {
        html += `
            <div class="col-md-4 mb-4">
                <div class="card bg-secondary text-white h-100 border-info">
                    <div class="card-body">
                        <h5 class="card-title text-info">${item.title}</h5>
                        <p class="card-text">${item.text}</p>
                        <h4 class="text-warning">${item.price.toLocaleString()} ₽</h4>
                        <button class="btn btn-outline-info w-100 mt-3"
                                onclick="window.navigateTo('detail', ${item.id})">
                            Подробнее
                        </button>
                    </div>
                </div>
            </div>
        `;
    });
    html += '</div>';
    this.parent.innerHTML += html;
}
